# Електронна бібліотека

## Опис функціоналу

Цей проєкт — консольний додаток для управління електронною бібліотекою. Програма дозволяє:

- Додавати книги до бібліотеки;
- Переглядати список усіх книг;
- Видаляти книги за їхнім ID;
- Додавати користувачів;
- Переглядати список користувачів;
- Видаляти користувачів за їхнім ID;
- Оформляти видачу книги користувачам (створення запису про позичання);
- Переглядати список виданих книг.

Проєкт побудований з дотриманням принципів чистого коду, що забезпечує зручність підтримки та розширення.

## Programming Principles

У цьому проєкті було дотримано наступних принципів програмування:

1. **DRY (Don't Repeat Yourself)** — код організовано так, щоб уникнути дублювання, наприклад, логіка збереження і завантаження даних винесена в окремий `FileHelper`.
2. **KISS (Keep It Simple, Stupid)** — простота і зрозумілість реалізації важливі для легкості підтримки.
3. **Single Responsibility Principle (SRP)** — кожен клас виконує лише одну задачу: `Repository` працюють з даними, `Service` — містять бізнес-логіку, а `Program` — керує взаємодією з користувачем.
4. **Open/Closed Principle (OCP)** — клас можна розширювати, не змінюючи існуючий код (наприклад, легко додати нові сервіси).
5. **Encapsulation** — приховання внутрішніх деталей реалізації через використання приватних полів і методів.

## Design Patterns

У проєкті використано такі патерни проєктування:

1. **Repository Pattern** — ізоляція логіки доступу до даних у окремих класах `BookRepository`, `UserRepository`, `LoanRepository`.  
   *Файли:* `Repositories/BookRepository.cs`, `Repositories/UserRepository.cs`, `Repositories/LoanRepository.cs`.

2. **Service Layer Pattern** — бізнес-логіка винесена у сервіси (`BookService`, `UserService`, `LoanService`) для розділення відповідальностей.  
   *Файли:* `Services/BookService.cs`, `Services/UserService.cs`, `Services/LoanService.cs`.

3. **Singleton (частково)** — через те, що репозиторії ініціалізуються один раз у сервісах, підтримується єдина точка доступу до даних під час виконання.

## Refactoring Techniques

Під час розробки програми застосовувалися такі техніки рефакторингу:

1. **Extract Method** — винесення повторюваних або довгих фрагментів коду у окремі методи для підвищення читабельності.
2. **Rename Variable** — використання зрозумілих, осмислених імен змінних і методів.
3. **Encapsulate Field** — використання приватних полів і публічних властивостей для захисту стану об'єктів.
4. **Introduce Parameter Object** — для методів, які працюють з кількома параметрами, використовували об’єкти (наприклад, модель Book), щоб уникнути передачі багатьох параметрів.
5. **Replace Magic Numbers with Constants** — константи для шляху збереження файлів, замість жорстко закодованих рядків.
